## 📋 プロジェクト基本情報

### プロジェクト名
`visiting-nursing-association-lp` / `jnhc-mra-lp_new`

### プロジェクトタイプ
**ランディングページ（LP）** - 有料会員募集用

### ビジネス目的
訪問看護ステーションの経営者・管理者向けに、実践的な経営ノウハウを提供する「スタンダード会員（月額10,000円）」の有料会員を募集する。

### ターゲット顧客
- **主要ターゲット**: 訪問看護ステーションの経営者・管理者
- **属性**: 開業済み、または開業準備中
- **課題**: 経営改善、財務管理、採用・定着、AI活用など

### 主要KPI
- **コンバージョン率（CV率）**: 最大化が最重要
- **離脱率**: 最小化
- **ページ滞在時間**: 適切な長さ（過度に長くしない）
- **スクロール深度**: 全セクション到達率

## 🏗️ プロジェクト構造の概要

### アーキテクチャパターン
- **コンポーネントベース**: React 19 + TypeScript
- **データ分離**: コンテンツデータは`src/content/`に集約
- **機能分離**: セクションコンポーネントは`src/components/sections/`に配置
- **レイアウト分離**: 共通レイアウトは`src/components/layout/`に配置

### 主要ディレクトリ
```
src/
├── components/     # UIコンポーネント（layout, sections, ui, legal）
├── content/        # テキストデータ・コンテンツ（分離管理）
├── shared/         # 共通リソース（constants, types）
└── App.tsx         # メインアプリケーション（セクション順序定義）
```

## 📊 セクション構成（CV最大化・黄金ルート）

### 現在のセクション順序（最適化済み）

1. **ProgressBar** - スクロール進捗表示

9. **Testimonials** - お客様の声（価格提示後の疑念解消）
10. **Profile** - 講師プロフィール（権威性の強化）
11. **FAQ** - よくある質問（14項目）

13. **LegalDocuments** - 法的表記
14. **Footer** - フッター

### セクション間の論理的流れ

```
認知（Hero） 
→ 共感（Problem） 
→ 理解（RoiSection, Weapons, ValueStack） 
→ 意思決定（Pricing）

```

## ⚙️ 技術要件

### 必須要件
- React 19.2.0
- TypeScript 5.8.2
- Vite 6.2.0
- Tailwind CSS v4 (CDN)
- レスポンシブデザイン（モバイル・タブレット・デスクトップ）

### パフォーマンス要件
- 初回読み込み時間: 3秒以内
- 画像最適化: FV以外はlazy loading
- コード分割: セクションごとにlazy loading
- バンドルサイズ: 最小化

## 📝 コンテンツ管理方針

### データ分離の原則
- **すべてのテキスト**: `src/content/*.ts`に集約
- **コンポーネント**: データを読み込むだけ（ハードコーディング禁止）
- **更新時**: コンテンツファイルのみ編集

### コンテンツファイル一覧
- `hero.ts` - Heroセクションのテキスト
- `problems.ts` - Problemセクションのテキスト
- `roi.ts` - ROI計算・比較データ
- `program.ts` - Weapons（4つの武器）の詳細
- `pricing.ts` - 料金・特典データ
- `faq.ts` - よくある質問（14項目）
- `closing.ts` - クロージングメッセージ
