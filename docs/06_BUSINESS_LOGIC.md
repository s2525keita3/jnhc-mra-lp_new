## 💼 ビジネス要件

### 主要機能

1. **有料会員の獲得**
   - 月額10,000円のスタンダード会員への入会促進
   - Stripe決済による自動決済

2. **価値の明確化**
   - 836万円相当の価値を月額10,000円で提供
   - セミナー月2回参加で元が取れる計算

3. **信頼構築**
   - 講師の実績と権威性の提示
   - お客様の声による社会的証明

4. **懸念の解消**
   - 14項目のFAQ
   - 30日間返金保証
   - 解約の容易さ

## 🎯 コンバージョンファネル

### ファネルステージ

```
認知 (Awareness)
  ↓ Heroセクション
興味 (Interest)
  ↓ Problem → RoiSection
検討 (Consideration)
  ↓ Weapons → ValueStack → Pricing
意思決定 (Decision)
  ↓ Testimonials → Profile → FAQ
行動 (Action)
  ↓ Closing → CTA
```

### 各ステージの目標

1. **認知**: 3秒以内に価値提案を理解
2. **興味**: 課題への共感と解決策への期待
3. **検討**: 提供価値の詳細理解
4. **意思決定**: 信頼構築と懸念解消
5. **行動**: CTAクリック → Stripe決済ページへ

## 💰 価格設定ロジック

### 料金体系

```typescript
{
  monthlyPrice: 10000,        // 月額（税別）
  monthlyPriceWithTax: 11000, // 月額（税込）
  yearlyPrice: 100000,        // 年額（税別）
  yearlyPriceWithTax: 110000  // 年額（税込）
}
```

### 価値スタック計算

```typescript
{
  totalValue: 836000,  // 総額相当価値
  monthlyPrice: 10000, // 月額会費
  multiplier: 83.6     // 倍率（836,000 ÷ 10,000）
}
```

### ROI計算ロジック

```typescript
// セミナー相場
seminarFee: 5000円/回
seminarsPerMonth: 2回
monthlyValue: 5000 × 2 = 10,000円

// 結論: セミナー月2回参加で月額会費と同額
```

## 🔄 ユーザーフロー

### 典型的なユーザージャーニー

1. **ランディング** (Hero)
   - 価値提案を理解
   - 統計数値で信頼性を確認

2. **課題認識** (Problem)
   - 現在の課題を確認
   - 解決策への期待

3. **価値理解** (RoiSection → Weapons → ValueStack)
   - ROI計算で合理性を理解
   - 提供価値の詳細を確認

4. **価格確認** (Pricing)
   - 月額10,000円を確認
   - 特典を確認

5. **信頼確認** (Testimonials → Profile)
   - お客様の声で安心
   - 講師の実績で権威性を確認

6. **懸念解消** (FAQ)
   - 14項目の質問で疑問を解消

7. **行動** (Closing)
   - 最終的なCTAクリック
   - Stripe決済ページへ遷移

## 🎨 インタラクション設計

### スクロール挙動

- **ProgressBar**: スクロール進捗を可視化
- **Header**: スクロール時に背景色変更
- **StickyCta**: 600pxスクロール後に表示

### ホバーエフェクト

- **カード**: 浮き上がる（`hover-lift`）
- **CTAボタン**: シャドウ強化、少し浮き上がる
- **リンク**: 色変更、下線表示

### アニメーション

- **フェードアップ**: セクション表示時
- **遅延アニメーション**: 複数要素を順番に表示
- **パルス**: 重要な要素（時計アイコンなど）

## 🔐 セキュリティ要件

### データ保護

- **個人情報**: 暗号化通信（SSL/TLS）
- **決済情報**: Stripeに委託（PCI DSS準拠）
- **環境変数**: `.env.local`で管理（Gitにコミットしない）

### 入力検証

- **フォーム**: 現在はStripe決済ページに委託
- **将来拡張時**: クライアント側・サーバー側両方で検証
